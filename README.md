# 1. 프로젝트 소개 
사용자의 일정을 월별 달력 형식으로 관리하고, 날짜별 할 일을 기록할 수 있는 웹 애플리케이션입니다.
Flask 프레임워크와 파이썬 자체 데이터베이스 SQLite를 결합하여 사용자 인증 및 실시간 데이터 관리를 구현했습니다.

# 2. 주요 기능
회원가입 및 로그인: Flask-Session을 이용한 사용자 인증 및 보안 로그인.

동적 달력 인터페이스: Jinja2 템플릿 엔진을 활용해 현재 날짜 및 할 일이 있는 날을 시각적으로 표시.

Todo CRUD: 특정 날짜의 할 일을 등록(Create), 조회(Read), 수정(Update), 삭제(Delete) 및 완료 체크 기능.

반응형 연도/월 선택: 상단 네비게이션을 통해 원하는 시점으로 빠른 이동 가능.

회원가입, 로그인 시 중복 아이디나 DB에 저장되어 있지 않은 아이디로 로그인 시 flash로 경고 알림이 뜨게 했습니다.

사용자가 할 일을 적었다면 적은 날에 작은 도트 점이 하나 뜨게 됩니다.

할 일 완료 시 체크박스를 누르게 되면 글자에 선이 그어지게 되고 작은 도트 점이 사라지게 됩니다.

할 일 목록에 적은 할 일 텍스트를 누르게 되면 수정이 가능하도록 설정 했습니다.

사용자가 다른 연도로 이동하고 난 후에 다시 오늘 날짜로 돌아올 수 있게끔 달력 왼쪽 상단 Today 버튼에 루트 /main 을 넣어서
현재 사용자가 보이는 페이지를 오늘 날짜로 보일 수 있게 했습니다.

오늘 날짜와 사용자가 선택한 날짜를 구분할 수 있게 오늘 날짜는 파랑, 선택한 날짜는 노랑으로 표시했습니다.

# 3. 기술 스택
Language: Python

Framework: Flask

Database: SQLite3

Frontend: HTML5, CSS3 (Flexbox/Grid), JavaScript (DOM Manipulation)

Template Engine: Jinja2

# 4. 핵심 기술 포인트 

DOM 조작을 통한 동적 수정: document.createElement를 사용하여 페이지 새로고침 없이 자바스크립트로 즉석에서 폼을 생성해 데이터를 전송하도록 구현했습니다.

SQL 쿼리 최적화: 날짜별 할 일 개수를 GROUP BY와 COUNT를 사용하여 한 번의 쿼리로 가져와 달력에 표시하는 효율적인 데이터 바인딩을 수행했습니다.

Jinja2 조건부 렌더링: 서버 사이드에서 달력의 요일 위치와 오늘 날짜 등을 계산하여 클라이언트에 최적화된 HTML을 제공합니다.

# 6. 문제 해결

1. 달력 요일 정렬 및 빈 칸 처리 오류

문제: 매달 1일이 무조건 일요일부터 시작하거나, 월말 이후 다음 달 날짜가 섞여 나오는 현상 발생.

원인: 서버에서 전달된 first_day_of_week(1일의 시작 요일) 값을 HTML 테이블 구조에 정확히 매칭하지 못함.

해결: Jinja2의 empty_days 리스트를 활용해 1일 이전의 칸을 빈 <td>로 먼저 채우고, (공백 + 현재 날짜) % 7 연산을 통해 토요일마다 정확히 줄바꿈(</tr><tr>)이 일어나도록 로직을 설계하여 해결했습니다.

2. 수정(Update) 시 데이터 유실 및 경로 오류

문제: 할 일을 수정할 때 어떤 데이터를 고쳐야 하는지 서버가 식별하지 못하거나, 수정 후 엉뚱한 날짜로 리다이렉트되는 문제.

원인: 수정 폼에 고유 id가 누락되었고, 수정 완료 후 현재 보고 있던 date 파라미터를 서버에 다시 전달하지 않아 발생.

해결: 자바스크립트의 document.createElement를 이용해 동적 폼을 생성하고, Action 주소에 id를 포함(/edit_todo/ID)시켰습니다.

3. 브라우저 자동완성으로 인한 레이아웃 깨짐

문제: 로그인/회원가입 페이지에서 브라우저의 '비밀번호 저장' 기능이 작동하면서 입력창 색상이 강제로 변하거나 원치 않는 값이 미리 채워짐.

원인: 브라우저의 기본 autocomplete 정책이 CSS 스타일보다 우선순위가 높음.

해결: 실제 입력창 위에 display: none; 처리된 더미 input 태그를 배치하여 브라우저의 자동완성 로직을 가로채고, autocomplete="new-password" 속성을 명시하여 깔끔한 UI를 유지하도록 해결했습니다.
